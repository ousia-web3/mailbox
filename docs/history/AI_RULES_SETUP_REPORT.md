# AI 협업 규칙 및 오류 관리 체계 구축 보고서

## 1. 개요

- **작업 일자**: 2026-01-15
- **작업 목표**: 프로젝트 전용 AI 작업 룰을 코드와 문서로 고정하고, 오류 재발을 최소화하기 위한 이력 관리 체계 구축
- **적용 대상**: 프로젝트 루트 및 문서 디렉토리

## 2. 생성된 파일 및 디렉토리

### 2.1 규칙 및 정책 (`rules/`, `.cursor/`)

| 파일명                    | 설명                                                                |
| ------------------------- | ------------------------------------------------------------------- |
| `.cursor/instructions.md` | **AI 작업 헌법**. 실행 제한, 최소 diff, 정적 검증 등 절대 규칙 정의 |
| `rules/EXEC_POLICY.md`    | **실행 권한 정책**. 사용자 명시적 요청 시에만 실행 허용             |
| `rules/CHANGELOG.md`      | **변경 이력**. 수정, 개선, 오류 조치 내역 기록                      |

### 2.2 로그 및 이력 (`logs/`)

| 파일명                  | 설명                                                   |
| ----------------------- | ------------------------------------------------------ |
| `logs/ERROR_HISTORY.md` | **오류 기록**. 발생 오류, 원인, 해결, 재발 방지책 기록 |

### 2.3 기타

- `.gitignore`: `logs/ERROR_HISTORY.md` 추적 허용 설정 추가
- `README.md`: "AI 협업 규칙 및 오류 관리" 섹션 추가

## 3. 주요 규칙 요약

### 실행 제한

- AI는 기본적으로 **명령어를 자동 실행하지 않음**
- 사용자가 "실행해"라고 명시한 경우에만 예외적으로 허용

### 오류 관리

- 오류 발생 또는 수정 시 `logs/ERROR_HISTORY.md` 필수 갱신
- 작업 시작 전 `ERROR_HISTORY.md` 및 `CHANGELOG.md` 선행 참조

### 코드 수정

- 최소 diff 원칙 준수
- 관련 없는 파일 수정 금지

## 4. 향후 절차

- 모든 AI 작업은 위 문서를 기준으로 수행
- 주기적인 문서 갱신을 통해 지식 자산화
