# 뉴스 상세 페이지 날짜 추출 로직 개선 보고서

## 개선 배경

구글 뉴스 검색 결과의 날짜 정보가 부정확하거나 누락되는 경우가 있어, 뉴스 상세 페이지에 직접 접속하여 정확한 발행일(입력, 수정, 업데이트 날짜 등)을 추출하고 이를 기반으로 필터링하도록 로직을 개선했습니다.

## 주요 변경 사항

`news_collector_working.py` 파일의 `extract_date_from_google_news` 메서드 내에 다음 로직을 추가했습니다:

1.  **다양한 날짜 선택자 추가**:

    - `.input-date`, `.reg-date`, `.art_date`, `.view_date`, `.info_date` 등 국내 뉴스 사이트에서 흔히 사용되는 날짜 요소 클래스를 추가했습니다.
    - `span.txt_info`, `.bar_info` 등 특정 포털이나 사이트의 구조를 반영했습니다.

2.  **텍스트 패턴 매칭 도입**:
    - HTML 구조에 의존하지 않고, 본문 텍스트 전체에서 날짜 관련 키워드와 패턴을 검색합니다.
    - `입력 2026-01-12`, `기사입력 2026.01.12`, `수정 ...`, `업데이트 ...` 등의 패턴을 정규표현식으로 찾아냅니다.
    - 이를 통해 "0입력", "정보", "데이터" 등 사용자가 언급한 키워드 주변의 날짜 정보도 효과적으로 포착할 수 있게 되었습니다.

## 기대 효과

- **정확도 향상**: 검색 결과 목록에 표시된 날짜보다 상세 페이지의 실제 기사 입력 시간을 우선적으로 반영하여, 더 정확한 "전날 뉴스" 필터링이 가능해집니다.
- **수집량 증가**: 날짜 형식이 특이하거나 메타 태그가 없는 사이트에서도 텍스트 패턴 매칭을 통해 유효한 날짜를 추출할 수 있어, 날짜 불명으로 버려지는 기사가 줄어듭니다.

## 검증

- 테스트 실행 결과, 상세 페이지에서 날짜를 성공적으로 추출하여 수집 목록에 추가하는 것을 확인했습니다.
