# 날짜 패턴 추가 적용 보고서

## 요청 사항

다음과 같은 날짜 패턴을 추가하여 뉴스 상세 페이지에서 날짜를 추출할 수 있도록 요청함:

1.  `2026-01-11` (YYYY-MM-DD)
2.  `2026.01.11` (YYYY.MM.DD)
3.  `2026. 01. 11` (YYYY. MM. DD, 공백 포함)

## 적용 내용

`news_collector_working.py` 파일의 두 가지 핵심 메서드에 해당 패턴을 추가했습니다.

### 1. `extract_date_from_content` (본문 날짜 추출)

- 기존 패턴에 `r'(\d{4})\.\s*(\d{1,2})\.\s*(\d{1,2})'`를 추가하여, 점(.) 뒤에 공백이 있거나 없는 경우를 모두 커버하도록 했습니다.

### 2. `extract_date_from_google_news` (구글 뉴스 상세 추출)

- `text_patterns` 리스트에 다음 3가지 단독 날짜 패턴을 추가했습니다:
  - `r'(\d{4}-\d{1,2}-\d{1,2})'`
  - `r'(\d{4}\.\d{1,2}\.\d{1,2})'`
  - `r'(\d{4}\.\s+\d{1,2}\.\s+\d{1,2})'`
- 또한 기존의 "입력", "수정" 등의 키워드가 포함된 패턴들도 공백 처리를 유연하게(`\s*`) 수정하여 다양한 표기 방식에 대응하도록 개선했습니다.

## 기대 효과

- "입력"이나 "수정" 같은 키워드 없이 날짜만 덩그러니 표기된 사이트에서도 날짜를 정확히 추출할 수 있게 됩니다.
- 특히 `2026. 01. 11`과 같이 띄어쓰기가 포함된 형식을 사용하는 언론사의 기사도 누락 없이 수집할 수 있습니다.

## 검증

- 테스트 스크립트 실행 결과, 구글 뉴스 수집 과정에서 에러 없이 정상적으로 동작함을 확인했습니다.
