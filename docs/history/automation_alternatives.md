# 윈도우 작업 스케줄러 대체 자동화 방안

로컬 컴퓨터의 상태(전원 꺼짐, 인터넷 연결 등)에 구애받지 않고 안정적으로 메일을 발송하기 위한 대체 방안들을 정리했습니다.

## 1. GitHub Actions (추천)

코드가 GitHub 저장소에 있다면 가장 간편하게 설정할 수 있는 무료(공개 저장소 기준) 클라우드 자동화 도구입니다.

### 장점

- **서버 불필요**: 별도의 서버를 구축하거나 비용을 지불할 필요가 없습니다.
- **설정 간편**: `.github/workflows` 폴더에 YAML 파일 하나만 추가하면 됩니다.
- **무료**: 공개 저장소는 무료이며, 비공개 저장소도 월 2,000분까지 무료 제공됩니다.

### 설정 방법 예시

`.github/workflows/email_schedule.yml` 파일을 생성합니다.

```yaml
name: Email Sender Cron

on:
  schedule:
    # 매일 오전 9시 (KST 기준 오후 6시) 실행
    # UTC 기준이므로 한국 시간 -9시간 설정 필요
    - cron: "0 0 * * *"
  workflow_dispatch: # 수동 실행 버튼 추가

jobs:
  send-email:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: "3.9"

      - name: Install dependencies
        run: |
          pip install -r requirements.txt

      - name: Run script
        env:
          # GitHub Repository Secrets에 등록된 환경변수 사용
          EMAIL_USER: ${{ secrets.EMAIL_USER }}
          EMAIL_PASS: ${{ secrets.EMAIL_PASS }}
        run: python main.py
```

## 2. 클라우드 서버 (AWS, Google Cloud, Azure)

24시간 켜져 있는 가상 컴퓨터(VM)를 임대하여 사용하는 방식입니다.

### 장점

- **완전한 제어**: 로컬 PC와 거의 동일한 환경을 구성할 수 있습니다.
- **안정성**: 전원이나 인터넷 끊김 걱정이 없습니다.

### 단점

- **비용**: 프리티어(무료 구간)를 넘어가면 비용이 발생합니다.
- **관리**: OS 업데이트, 보안 설정 등을 직접 관리해야 합니다.

### 추천 서비스

- **AWS EC2 (Free Tier)**: 1년간 t2.micro 또는 t3.micro 인스턴스 무료 사용 가능.
- **Google Cloud Compute Engine**: e2-micro 인스턴스 등 조건부 무료 제공.
- **Oracle Cloud**: 평생 무료 인스턴스 제공 (가용성 구역에 따라 다름).

## 3. 서버리스 (AWS Lambda, Google Cloud Functions)

서버를 관리하지 않고 코드만 올려두면 특정 시간(EventBridge/Cloud Scheduler)에 실행되는 방식입니다.

### 장점

- **비용 효율**: 실행된 시간만큼만 과금되므로 간헐적인 작업에 매우 저렴합니다.
- **관리 불필요**: 서버 관리가 필요 없습니다.

### 단점

- **설정 복잡**: 코드를 함수 형태로 변경해야 할 수 있으며, 패키징 과정이 필요합니다.
- **실행 시간 제한**: 보통 15분 내외의 실행 시간 제한이 있습니다.

---

## 4. 소형 홈 서버 (NAS, 라즈베리파이)

집이나 사무실에 24시간 켜두는 저전력 장비를 이용하는 방법입니다.

### 장점

- **데이터 로컬 유지**: 민감한 데이터를 클라우드에 올리지 않아도 됩니다.
- **추가 비용 없음**: 장비가 이미 있다면 전기세 외 추가 비용이 없습니다.

### 활용 예시

- **Synology NAS**: 작업 스케줄러 기능 내장. 파이썬 스크립트 실행 가능.
- **Raspberry Pi**: 리눅스 기반의 `cron`을 사용하여 예약 작업 설정.

## 요약 및 추천

| 구분          | GitHub Actions                                  | 클라우드 서버 (EC2 등)                         | 서버리스 (Lambda)                                | 홈 서버 (NAS/Pi)              |
| :------------ | :---------------------------------------------- | :--------------------------------------------- | :----------------------------------------------- | :---------------------------- |
| **난이도**    | 하                                              | 중                                             | 상                                               | 중                            |
| **비용**      | 무료 (조건부)                                   | 유료 (프리티어 가능)                           | 매우 저렴                                        | 장비 구매 비용                |
| **안정성**    | 높음                                            | 매우 높음                                      | 높음                                             | 중간 (정전 등)                |
| **추천 대상** | **코드 관리와 자동화를 한 번에 하고 싶은 경우** | **기존 윈도우 환경을 그대로 옮기고 싶은 경우** | **개발 지식이 있고 비용을 최소화하고 싶은 경우** | **이미 NAS 장비가 있는 경우** |
